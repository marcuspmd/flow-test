    Class VariableService

    Service responsible for variable interpolation and resolution

    Manages the hierarchical context of variables and provides functionality for interpolation using the {{variable_name}} syntax. Supports multiple scopes: runtime > suite > imported > global.

    Example

    const variableService = new VariableService({
      global: { api_url: 'https://api.example.com' },
      suite: { user_id: 123 },
      runtime: { auth_token: 'abc123' },
      imported: { auth: { token: 'xyz789' } }
    });

    const url = variableService.interpolate('{{api_url}}/users/{{user_id}}');
    // Results in: 'https://api.example.com/users/123'

    const authToken = variableService.interpolate('{{auth.token}}');
    // Results in: 'xyz789' (imported variable)

    interpolate

    • interpolate(template: any): any

      Interpolates variables in a string or complex structure

      Replaces {{variable_name}} placeholders with corresponding values following the scope hierarchy: runtime > suite > imported > global. Supports interpolation in strings, arrays and objects recursively.

      Parameters

      • template: any

        String, array or object containing variable placeholders

      Returns any

      Value with all variables interpolated

      Example

      // Simple string
      interpolate('Hello {{username}}!') // → 'Hello john!'

      // Complex object
      interpolate({
        url: '{{api_url}}/users/{{user_id}}',
        headers: { 'Authorization': 'Bearer {{token}}' }
      })
      // → { url: 'https://api.com/users/123', headers: { 'Authorization': 'Bearer abc123' } }

      // Array
      interpolate(['{{env}}', '{{version}}']) // → ['production', '1.0.0']

    PrivateresolveVariable

    • resolveVariable(variablePath: string): any
      Private

      Resolves a variable following the scope hierarchy

      Searches for a variable in precedence order: runtime > suite > imported > global > globalRegistry. Supports dot notation for exported variables (ex: auth.token).

      Parameters

      • variablePath: string

        Variable path (can use dot notation)

      Returns any

      Variable value or undefined if not found

      Example

      resolveVariable('username') // Searches in all scopes
      resolveVariable('auth.token') // Searches specifically for exported variable from nodeId 'auth'
